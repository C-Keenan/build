Game Server Helper

Overview

'Game Server Helper' is a linux command-line script interface designed to run various popular game servers such as "Minecraft", "Veloren", "Satisfactory", and "Factorio" by utilizing the 'Docker Compose' program.

Installation

If you run the install script it will first ask if you would like to install the command-line scripts, which entails adding an alias to the current user's '.bashrc' file directed to the installation directory `/usr/gsh/script-files/gsh`, it will then copy the 'gsh' folder, which includes the default docker-compose.yml files for the various game servers distributed with the script, into the linux 'usr' directory and then pull the necessary linux packages "docker-ce", "docker-ce-cli", "containerd.io", "docker-buildx-plugin", and "docker-compose-plugin". It then asks you if you would like to test the docker installation, which is not usually necessary, and run the `docker run hello-world` command, which is the 'Docker' test image, to make sure that the installation is up and running correctly. The next step is to check whether or not the system has the required 'colrm' utility utilized by the alias command `gsh factorio down` which checks the factorio 'server-settings.json' file for the autosave interval and runs a timer to allow an autosave to happen so the user does not lose any recent game progress which might occur if the container is brought down immediately which can still be done by running the alias command `gsh factorio down --now`. The installation script will then check to see if the environment directory, used by the donation message function of the script, is in existance on the system and copy the necessary environment files into said directory. There are also default environment files for each of the various game servers that are packaged along-side the script which are necessary for the servers to function as expected by the script that are copied into the respective docker container folders at this point in the script. The script will then print a closing message thanking the user for installing the 'Game Server Helper' script and directing them to provide feature requests to the email address specified in the installation script. The installation script can be run as the 'root' user or a general user with 'sudo' privledges, and should be capable of running on debian linux systems that do not contain the 'sudo' package. If you answer "no" to the installation script asking if you would like to install 'Game Server Helper' then the script will first print a message thanking the user for running the script and mentioning that since they answered "no" the script will now exit and reminding them to "Have a great day!"
